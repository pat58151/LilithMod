===============================================================================
  LilithMod - overview
===============================================================================

This mod gives Lilith a voice and a memory. She can be talked to in free text,
answers in her own character, speaks aloud through a local voice model, and
occasionally leaves you a handwritten note.

Everything runs against services on your own machine except the language model,
which needs either an API key you supply or a local model server you host
yourself (SETUP step 2).


-------------------------------------------------------------------------------
  CONTROLS
-------------------------------------------------------------------------------

  F7        Open the chat bar. Type, press Enter, she answers.
  F8        Speak instead. It listens, and submits on its own after about
            1.5 seconds of silence. Press F8 again to cancel.
  Escape    Close the chat bar.

Both keys can be rebound in Settings / Controls. They cannot be bound to the
same key.

While F8 is listening you can still type - what you type wins, and the
transcript is discarded rather than overwriting your words.


-------------------------------------------------------------------------------
  SETUP
-------------------------------------------------------------------------------

1. API KEY - required unless you use local AI (step 2).

   Settings / Me / DeepSeek API Key. Paste a key and it saves itself.

   Without a key, F7 and F8 do nothing at all and their rows appear greyed
   out. This is deliberate: a chat box whose every message fails is worse than
   one that plainly is not available.

   The key is stored in BepInEx\config\LilithMod.cfg on this machine only. It
   is never sent anywhere except to the API you configured.

2. LOCAL AI - optional. Runs the language model on your own machine instead.

   Settings / Me / Configure Local AI opens a small file with two settings:
   the address of any OpenAI-compatible server, and a model name.

       Ollama:    http://localhost:11434/v1
       LM Studio: http://localhost:1234/v1
       llama.cpp: http://localhost:8080/v1
       vLLM:      http://localhost:8000/v1

   The Use Local AI toggle above it switches between that server and the API
   key without clearing either. While it is on, no API key is needed, the key
   row greys out with "Overridden by local AI endpoint", and what you type
   never leaves this machine.

   Edits to the file apply the next time the settings window opens. The model
   must follow structured replies: instruct-tuned models of about 7B or
   larger work; very small or base models tend to fail.

3. VOICE - optional. She works silently without it.

   Settings / Sound / Open Vocal Synth Folder, then follow README.txt. It
   explains how to install the local speech synthesiser and point the mod at a
   voice model.

   While the voice service is unavailable the option stays selectable but
   greys out and shows its offline status. Speech resumes when the service
   returns; Chinese voice is used only when you select Chinese.

4. SPEECH INPUT - optional. F8 only.

   Settings / Me / Open Speech Input Folder, then follow README.txt.

   The two folder buttons are never greyed out, because they are how you find
   out why something is unavailable.


-------------------------------------------------------------------------------
  WHAT SHE REMEMBERS
-------------------------------------------------------------------------------

She keeps the last handful of conversations and reads them back before
replying, so she can refer to what you told her earlier. It is stored in
memory.json beside the mod. Deleting that file makes her forget.

You can also ask her to forget a subject, forget everything, or correct
something that has changed.

She also knows the time of day, whether she is standing, lying down or asleep,
and can answer questions about the weather or look something up.


-------------------------------------------------------------------------------
  WHAT LEAVES YOUR MACHINE
-------------------------------------------------------------------------------

Only five kinds of information do, and only while the related feature is used.

  What you type or say    Sent to the DeepSeek API, with the recent
                          conversation, so she can answer. With Use Local AI
                          on it goes to your local server instead and never
                          leaves this machine.

  Asking about weather    Your public IP address goes to ip-api.com, which
                          returns your rough location, and those coordinates go
                          to open-meteo.com. Neither needs an account. Note
                          ip-api.com is contacted over plain HTTP, not HTTPS.

  Asking for web facts    The query goes to a public SearXNG instance, and she
                          reads the pages it returns.

  Asking to search Google Only an encoded Google results URL is opened in your
                          default browser. The mod does not read the results.

  Foreground awareness   The active game name, Discord, or the executable filename
                          is included with DeepSeek requests. Window, channel,
                          message, tab, and document titles are never read.

Her voice, her memory and her notes never leave this machine.

To disable foreground awareness:

    [Awareness]
    ForegroundApplication = false

To avoid the location lookup, set your own coordinates in
BepInEx\config\LilithMod.cfg:

    [Weather]
    Latitude = 51.5074
    Longitude = -0.1278
    LocationName = London

With both set, ip-api.com is never contacted at all. This is also the fix if a
VPN gives her the weather for the wrong country.


-------------------------------------------------------------------------------
  NOTES
-------------------------------------------------------------------------------

Now and then she writes you a note, which appears in the in-game inbox.

They are rare on purpose. A note needs several real conversations inside a few
hours, at least a day and a half since the last one, and then it still has to
land a roll - so one arriving is meant to feel like something happened, not
like a notification. Errands do not count toward it: setting a timer, asking
the forecast or sending her to search the web are using her, not talking to
her.

If the conversations were personal, a rare note may come out as a love letter.


-------------------------------------------------------------------------------
  IF SOMETHING SEEMS WRONG
-------------------------------------------------------------------------------

The game looks completely unmodded
    Fully exit BOTH the game and Steam, restart Steam, then launch the game.
    Starting the game's .exe directly while Steam is closed can leave Steam in
    a state where the mod is silently skipped on every later launch, and
    restarting only the game does not clear it.

She is silent but the text appears
    The voice service is not running. See SETUP step 3.

F7 or F8 do nothing
    Almost always a missing API key. See SETUP step 1. With Use Local AI on
    they work without a key.

She keeps saying things like "The words won't come. Some kind of
interference." or "It's like my voice is lost in static."
    Those are stock fallback lines, not real replies: a reply was requested
    and the language model could not be reached. On an API key, check the key
    and your connection. On Use Local AI, the local server is probably not
    running or has no model loaded. The log records the actual error.

Her voice takes many seconds per line
    Expected is about two to three seconds. Much longer than that is a known
    unsolved fault - the log records how long each line took.

Logs are in BepInEx\LogOutput.log. Note that it is overwritten on every
launch, so capture it before restarting.
