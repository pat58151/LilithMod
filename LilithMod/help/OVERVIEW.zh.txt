===============================================================================
  LilithMod - 概览
===============================================================================

这个MOD给了莉莉丝声音和记忆。你可以用自由文字和她说话，她会以自己的性格回答，
通过本地语音模型出声说话，并且偶尔会给你留下一封手写的信。

除了语言模型之外，一切都运行在你自己的电脑上。只有语言模型需要你自己提供的
API密钥。


-------------------------------------------------------------------------------
  操作
-------------------------------------------------------------------------------

  F7        打开聊天栏。输入后按Enter，她就会回答。
  F8        改用说话。它会聆听，并在大约1.5秒的静默后自动提交。再按一次F8可以
            取消。
  Escape    关闭聊天栏。

两个按键都可以在 设置 / 操作 中重新绑定。它们不能绑定到同一个键。

F8聆听期间你仍然可以打字——你打的字优先，听到的语音会被丢弃，而不会覆盖你写的
内容。


-------------------------------------------------------------------------------
  设置
-------------------------------------------------------------------------------

1. API密钥 - 以下所有功能都需要它。

   设置 / Me / DeepSeek API Key。粘贴密钥后会自动保存。

   没有密钥时，F7和F8完全没有反应，对应的行会显示为灰色。这是故意的：一个每条
   消息都必定失败的聊天框，比一个明确显示不可用的聊天框更糟。

   密钥只保存在本机的 BepInEx\config\LilithMod.cfg 中。除了你所配置的API之外，
   它不会被发送到任何地方。

2. 语音 - 可选。没有它她也能安静地工作。

   设置 / 声音 / 打开合成语音文件夹，然后按照 README.txt 操作。里面说明了如何
   安装本地语音合成器，以及如何让MOD找到语音模型。

   语音服务不可用时，该选项仍可选择，但会变灰并显示离线状态。服务恢复后会自动
   恢复语音合成。只有明确选择中文时才会使用中文语音。

3. 语音输入 - 可选。仅用于F8。

   设置 / Me / 打开语音输入文件夹，然后按照 README.txt 操作。

   两个打开文件夹的按钮永远不会变灰，因为它们正是你查明某项功能为何不可用的
   途径。


-------------------------------------------------------------------------------
  她记得什么
-------------------------------------------------------------------------------

她会保留最近的若干次对话，并在回答之前读一遍，所以她能提到你之前告诉过她的事。
这些内容保存在MOD旁边的 memory.json 中。删除该文件她就会忘记。

你也可以直接让她忘记某个话题、忘记全部内容，或者更正已经改变的事情。

她还知道现在是一天中的什么时候、自己是站着、躺着还是睡着，也可以回答天气或者
帮你查找资料。


-------------------------------------------------------------------------------
  哪些信息会离开你的电脑
-------------------------------------------------------------------------------

只有以下四类信息会离开本机。

  你输入或说出的内容    连同最近的对话一起发送到 DeepSeek 的 API，用来生成
                        回复。

  询问天气时            你的公网 IP 地址会发送到 ip-api.com，它返回你的大致
                        位置，这些坐标再发送到 open-meteo.com。两者都不需要
                        账号。注意 ip-api.com 使用的是普通 HTTP，而非 HTTPS。

  让她搜索时            搜索词会发送到公开的 SearXNG 实例，她会阅读返回的
                        页面。

  前台应用感知          启用时，游戏名称、Discord 或可执行文件名会随请求发送到
                        DeepSeek API。不会读取窗口、频道、消息、标签页或文档标题。

她的声音、记忆和信件从不离开这台电脑。

如需关闭前台应用感知：

    [Awareness]
    ForegroundApplication = false

如果不想让位置被自动检测，可以在 BepInEx\config\LilithMod.cfg 中设置自己的
坐标：

    [Weather]
    Latitude = 39.9042
    Longitude = 116.4074
    LocationName = 北京

两项都设置后，就完全不会连接 ip-api.com。如果 VPN 让她报出错误国家的天气，
这也是解决办法。


-------------------------------------------------------------------------------
  信件
-------------------------------------------------------------------------------

她偶尔会给你写一封信，出现在游戏内的收件箱里。

信很少见是故意的。一封信需要在几个小时之内有过几次真正的对话，距离上一封至少
一天半，然后还要通过一次随机判定——这样收到信时才像是发生了什么，而不像一条
通知。跑腿的事不计入其中：设定计时器、问天气或者让她搜索网页，那是在使用她，
而不是在和她说话。

如果这些对话足够私人，那么这封罕见的信可能会写成一封情书。


-------------------------------------------------------------------------------
  如果有什么不对劲
-------------------------------------------------------------------------------

游戏看起来完全没有被修改
    彻底退出游戏和Steam，重新启动Steam，然后再启动游戏。在Steam关闭时直接运行
    游戏的.exe，可能会让Steam进入某种状态，导致之后每次启动都静默跳过MOD，而
    只重启游戏无法解除。

有文字但她不出声
    语音服务没有在运行。见设置第2步。

F7或F8没有反应
    几乎总是缺少API密钥。见设置第1步。

她说一句话要好几秒
    正常大约是两到三秒。明显更久是一个已知的未解决故障——日志会记录每一句话
    所花的时间。

日志位于 BepInEx\LogOutput.log。注意它在每次启动时都会被覆盖，所以请在重启前
先保存。
