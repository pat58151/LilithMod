===============================================================================
  LilithMod - 概要
===============================================================================

このMODはリリスに声と記憶を与えます。自由な文章で話しかけることができ、彼女は
自分の性格のままに答え、ローカルの音声モデルで声に出して話し、ときどき手書きの
手紙を残します。

言語モデル以外はすべてお使いのPC上のサービスで動きます。言語モデルは、ご自分で
用意したAPIキーか、ご自身のPCで動かすローカルのモデルサーバー（セットアップの
2）のどちらかが必要です。


-------------------------------------------------------------------------------
  操作
-------------------------------------------------------------------------------

  F7        チャット欄を開きます。入力してEnterを押すと彼女が答えます。
  F8        声で話しかけます。約1.5秒黙ると自動で送信されます。もう一度F8を押す
            と取り消せます。
  Escape    チャット欄を閉じます。

どちらのキーも 設定 / 操作 で変更できます。同じキーには割り当てられません。

F8で聞き取り中でも文字入力はできます。その場合は入力した文字が優先され、聞き取
った音声は破棄されます。


-------------------------------------------------------------------------------
  セットアップ
-------------------------------------------------------------------------------

1. AIサービス - 彼女が話す言語モデルを選びます。

   設定 / Me / Configure AI Service を選ぶと、設定が2つだけの小さなファイルが
   開きます。OpenAI互換エンドポイントのアドレスと、モデル名です。初期状態では
   DeepSeekが設定されています。DeepSeek、OpenAI、OpenRouter、Groq、Mistral、
   xAI、Gemini、Together、Moonshot、Qwenなどのホスト型サービスがそのまま使え、
   ローカルサーバーも使えます。

       Ollama:    http://localhost:11434/v1
       LM Studio: http://localhost:1234/v1
       llama.cpp: http://localhost:8080/v1
       vLLM:      http://localhost:8000/v1

   ファイルの編集は、次に設定画面を開いたときに反映されます。モデルは構造化
   された返答に従える必要があります。7B以上のinstruct系モデルなら動作します
   が、ごく小さいモデルやbaseモデルは失敗しがちです。

2. APIキー - ホスト型サービスに必要です。ローカルサーバーには不要です。

   設定 / Me / API Key。設定したサービスのキーを貼り付けると自動的に保存され
   ます。ローカルサーバー利用時はキーの行が灰色になり「Not needed for local
   endpoint」と表示され、入力した内容がこのPCの外に出ることはありません。

   必要なキーがないとF7もF8もまったく動作せず、その行は灰色で表示されます。
   これは意図的なものです。送信するたびに必ず失敗するチャット欄は、はじめから
   使えないと分かっている方がましだからです。

   キーはこのPCの BepInEx\config\LilithMod.cfg にのみ保存されます。設定した
   API以外のどこにも送信されません。

3. 音声 - 任意です。なくても彼女は静かに動作します。

   設定 / サウンド / 音声合成フォルダを開く を選び、README.txt に従ってくださ
   い。ローカルの音声合成のインストール方法と、MODに音声モデルを認識させる方法
   が書かれています。

   音声サービスが使えない間もこの項目は選択できますが、灰色になりオフライン状態
   を表示します。サービスが復帰すると音声合成が自動的に再開します。中国語音声は
   中国語を選択した場合にのみ使用されます。

4. 音声入力 - 任意です。F8のみで使います。

   設定 / Me / 音声入力フォルダを開く を選び、README.txt に従ってください。

   フォルダを開く2つのボタンだけは決して灰色になりません。何が使えないのかを調
   べる手段そのものだからです。


-------------------------------------------------------------------------------
  彼女が覚えていること
-------------------------------------------------------------------------------

彼女は直近の何回かの会話を保持していて、返事をする前に読み返します。そのため、
以前に話したことに触れることができます。内容はMODと同じ場所の memory.json に保
存されます。このファイルを削除すると彼女は忘れます。

忘れてほしい話題やすべての記憶を会話の中で伝えたり、変わったことを訂正したり
することもできます。

さらに、今が一日のうちのいつか、自分が立っているか横になっているか眠っているか
も分かっていて、天気を答えたり調べものをしたりもできます。


-------------------------------------------------------------------------------
  このPCの外に出る情報
-------------------------------------------------------------------------------

外部に送られる情報は次の4種類だけです。

  入力した文章や音声    最近の会話とともに設定したAIサービスへ送られ、返事
                        が作られます。ローカルサーバー利用時はこのPCの外に
                        は出ません。

  天気を尋ねたとき      あなたのグローバルIPアドレスが ip-api.com に送られ、
                        おおよその位置が返り、その座標が open-meteo.com へ
                        送られます。どちらもアカウントは不要です。なお
                        ip-api.com との通信はHTTPSではなくHTTPです。

  検索を頼んだとき      検索語が公開のSearXNGインスタンスへ送られ、返って
                        きたページを彼女が読みます。

  前面アプリの認識      有効時、ゲーム名、Discord、または実行ファイル名が
                        AIサービスへ送られます。ウィンドウ、チャンネル、
                        メッセージ、タブ、文書のタイトルは読みません。

彼女の声、記憶、手紙がこのPCの外に出ることはありません。

前面アプリの認識を無効にするには：

    [Awareness]
    ForegroundApplication = false

位置の自動判定を避けたい場合は、BepInEx\config\LilithMod.cfg に自分の座標を
設定してください。

    [Weather]
    Latitude = 35.6762
    Longitude = 139.6503
    LocationName = 東京

両方を設定すると ip-api.com には一切接続しません。VPNのせいで違う国の天気を
答える場合の対処法でもあります。


-------------------------------------------------------------------------------
  手紙
-------------------------------------------------------------------------------

ときどき彼女は手紙を書き、ゲーム内の受信箱に届きます。

めったに届かないのは意図的です。数時間のうちに中身のある会話が何度か必要で、前
回から少なくとも1日半が経っていて、そのうえで抽選に当たらなければなりません。
届いたときに「何かが起きた」と感じられるように、通知のようにならないようにして
います。用事は会話として数えません。タイマーを設定する、天気を聞く、検索を頼む
といったことは、彼女を「使っている」のであって「話している」のではないからです。

会話が個人的なものだった場合、まれにその手紙がラブレターになることがあります。


-------------------------------------------------------------------------------
  うまく動かないとき
-------------------------------------------------------------------------------

MODがまったく適用されていないように見える
    ゲームとSteamの両方を完全に終了し、Steamを起動し直してから、ゲームを起動し
    てください。Steamを閉じたままゲームの.exeを直接起動すると、その後の起動でも
    MODが黙って読み込まれない状態がSteamに残ることがあります。ゲームだけを再起
    動しても解消しません。

文字は出るのに声が出ない
    音声サービスが動いていません。セットアップの3を参照してください。

F7やF8が反応しない
    ほとんどの場合APIキーがありません。セットアップの2を参照してください。
    ローカルサーバー利用時はキーなしで動きます。

「The words won't come. Some kind of interference.」のような決まった言葉を
繰り返す
    これは本当の返事ではなく、用意された代替のセリフです。返事を作ろうとし
    て言語モデルに接続できなかったことを意味します。ホスト型サービス利用時
    はキーと通信を確認してください。ローカルサーバー利用時は、サーバーが起
    動していないか、モデルが読み込まれていない可能性が高いです。実際のエラ
    ーはログに記録されています。

1行の発声に何秒もかかる
    通常は2〜3秒程度です。それより大幅に長い場合は既知の未解決の不具合です。
    各行にかかった時間はログに記録されています。

ログは BepInEx\LogOutput.log にあります。起動のたびに上書きされるので、再起動す
る前に保存してください。
